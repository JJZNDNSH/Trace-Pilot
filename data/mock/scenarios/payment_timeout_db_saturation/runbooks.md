# Payment Timeout During DB Saturation
service: payment-service

## 场景说明
用于模拟 payment-service 在交易高峰期因数据库资源饱和导致支付超时的故障。

## 识别特征
- 支付超时率升高，同时数据库连接池使用率逼近上限。
- 数据库锁等待和 CPU 使用率同步升高。
- 上游用户侧主要感知为支付提交超时或 504。

## 排查步骤
1. 检查 payment-service 和 payment-db 的连接池、锁等待与 CPU 指标。
2. 结合近期配置变更，确认连接池上限是否被调整。
3. 抽样查看超时请求日志，确认超时是否集中发生在数据库持久化阶段。

## 处置建议
1. 优先评估恢复连接池上限或限流保护，缓解数据库饱和。
2. 如果锁等待异常升高，检查是否存在慢 SQL 或热点事务。
3. 记录峰值流量、连接池占用率和变更时间，便于后续容量评估与复盘。
