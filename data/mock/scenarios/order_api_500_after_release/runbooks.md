# Order API 500 After Release
service: order-api

## 场景说明
用于模拟生产环境中 order-api 在发布后短时间内出现 500 激增的回归故障。

## 识别特征
- 告警在发布后 10 分钟内触发。
- 日志集中出现相同错误码 `ORDER_SUMMARY_KEY_MISSING`。
- 500 错误率与请求延迟同时上升。

## 排查步骤
1. 对齐最近 30 分钟发布记录与告警触发时间，确认是否存在明显时间相关性。
2. 检查 `POST /api/orders` 的错误日志，关注是否出现新版本特有的异常模式。
3. 对比新旧版本配置或特性开关，确认是否关闭了兼容性兜底逻辑。

## 处置建议
1. 如果错误集中出现在新版本且影响用户下单，优先准备受控回滚方案。
2. 如果存在特性开关兜底路径，可先评估关闭新逻辑的风险和收益。
3. 回滚前记录当前错误率、发布版本和受影响时间窗，便于后续复盘。
